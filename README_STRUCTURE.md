# Project Structure

This document provides a detailed overview of the project structure, dependencies, and application modules. It is intended to be a guide for developers working on this project.

## Directory Structure

The project follows a standard Laravel directory structure, with some additional directories for specific functionalities.

- **`app/`**: This is the core of the application.
    - **`Console/`**: Contains the Artisan commands for the application.
    - **`Exports/`**: Houses the classes responsible for exporting data to different formats, likely used by `maatwebsite/excel`.
    - **`Helpers/`**: Includes helper functions used throughout the application.
    - **`Http/`**: This directory contains the controllers, middleware, and form requests.
        - **`Controllers/`**: The controllers are organized into subdirectories based on their functionality.
    - **`Jobs/`**: Contains the queueable jobs for the application.
    - **`Models/`**: This directory holds all the Eloquent models.
    - **`Observers/`**: Contains observer classes that listen for events on Eloquent models.
    - **`Providers/`**: This directory contains all the service providers for the application.
    - **`Services/`**: Houses service classes that encapsulate business logic.
- **`bootstrap/`**: Contains the application's bootstrapping scripts.
- **`config/`**: This directory contains all the application's configuration files.
- **`database/`**: This directory contains the database migrations, seeds, and factories.
- **`public/`**: This is the document root for the application. It contains the `index.php` file, which is the entry point for all requests.
- **`resources/`**: This directory contains the views, raw assets (like SASS and JavaScript), and language files.
- **`routes/`**: This directory contains all the route definitions for the application, which are separated into multiple files based on their functionality.
- **`storage/`**: This directory contains the compiled Blade templates, file-based sessions, file caches, and other files generated by the framework.
- **`tests/`**: This directory contains the automated tests.
- **`vendor/`**: This directory contains the Composer dependencies.

## Dependencies

The following is a list of the main Composer dependencies used in this project:

- **`barryvdh/laravel-dompdf`**: A Laravel wrapper for the DOMPDF HTML to PDF converter.
- **`darkaonline/l5-swagger`**: A Laravel wrapper for Swagger UI and OpenAPI documentation.
- **`laravel/framework`**: The core Laravel framework.
- **`laravel/reverb`**: A first-party WebSocket server for Laravel applications.
- **`laravel/sanctum`**: A lightweight authentication system for SPAs (single-page applications), mobile applications, and simple, token-based APIs.
- **`laravel/socialite`**: An official Laravel package that provides a simple, fluent interface to OAuth authentication with providers like Facebook, Twitter, Google, GitHub, and Bitbucket.
- **`laravel/tinker`**: A powerful REPL for the Laravel framework.
- **`maatwebsite/excel`**: A popular Laravel package for importing and exporting Excel and CSV files.
- **`mcamara/laravel-localization`**: A package to handle localization in Laravel.
- **`phpoffice/phpword`**: A library for reading and writing Word documents.
- **`rap2hpoutre/fast-excel`**: A lightweight and fast Excel import/export package for Laravel.
- **`spatie/laravel-permission`**: A package to manage user permissions and roles in a Laravel application.

## Application Modules

The application is divided into several modules, each with its own set of controllers and routes.

- **API (`Api/`)**: This module contains the controllers for the application's API.
- **Authentication Management (`AuthManagement/`)**: This module handles user authentication, registration, and password management.
- **Dashboard Management (`DashboardManagement/`)**: This module is responsible for the application's dashboard.
- **Dental Management (`DentalManagement/`)**: This module contains the core logic for the dental management system.
- **Download Management (`DownloadManagement/`)**: This module handles file downloads.
- **System Management (`SystemManagement/`)**: This module provides system-level functionalities, such as user and role management.
